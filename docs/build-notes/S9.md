# S9 — Verifier 与策略注册表

- 新增 `config/strategy-registry.yaml`，默认策略均为 `candidate`，不可进入 Signal Loop。
- registry 是后续 Signal Loop 的准入源，只有 `status=approved` 且 max notional > 0 才允许交易。
- verifier 读取 walk-forward 报告和 `strategies.yaml` 阈值，输出 approved/rejected 与原因。
- 检查项包括 OOS 衰减、最大回撤、交易次数、扣费后有效、最近 OOS 未失效。
- maker/checker 强制分离：`maker_id == verifier_id` 直接拒绝。
- 被拒结果可写入 JSONL 日志，原因可审计。
- S9 不改变基础策略为 approved；S8 smoke 表现差，默认保持 candidate/rejected。
- S9 不下单、不改风控、不扩大交易标的。
