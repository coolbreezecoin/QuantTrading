# S1 — 配置加载与校验

- 新增 `crypto_quant_loop.config` 包，集中定义配置 schema 与加载入口。
- 使用 Pydantic v2，所有模型默认 `extra="forbid"`，避免拼错字段静默生效。
- YAML 读取只解析本地文件，不读取环境变量值，密钥仍只以 env var 名称存在。
- 单文件 schema 负责必填项、枚举、数值范围与基础结构。
- `load_all_configs()` 加载五个权威配置文件，并做轻量跨文件一致性检查。
- 跨文件检查包括 live/research 标的一致、策略订单类型与成交费用映射一致。
- 非法配置统一抛 `ConfigLoadError`，错误消息包含文件名与校验原因。
- S1 不调用交易所、不覆盖 exchange filters、不修改风控红线。
