# S11 — Paper Broker 与订单状态机

- Paper broker 是本地模拟 broker，不连接交易所、不使用真实 API key。
- 订单以 `client_order_id` 为幂等键，重复提交返回原订单，不改变仓位。
- Market buy 按 ask book 逐档撮合；limit buy 只有 ask 触及限价才成交。
- 深度不足时订单进入 `partially_filled`，可继续用后续 snapshot 补成交。
- 成交更新 position 平均价；卖出成交产生 realized PnL。
- 状态可保存为 JSON 并恢复，恢复后以持久化账本为准。
- reconciliation 会清理不在账本 open 集合中的悬挂订单。
- S11 只覆盖 paper，live adapter 留给 S14 门禁脚手架。
