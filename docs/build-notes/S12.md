# S12 — Loops 运行时

- 新增 `LoopSpec`、`LoopRuntime`，本地可用 `run_once()` 按优先级执行 loop。
- 每次 loop 运行写 heartbeat JSON，并追加 `loop-run-log.jsonl`。
- Dead-man switch 根据 `cadence_s * miss_factor` 判断 stale loop。
- stale loop 只产生本地 `DeadmanAlert` 与保守动作描述，不发外部告警。
- 新增 APScheduler adapter，可把同一组 LoopSpec 注册为 interval jobs。
- 测试覆盖优先级、heartbeat/run-log、dead-man stale 检测。
- S12 不启动常驻后台进程；后续部署时显式调用 scheduler。
