# S3 — 数据层：Data Health Loop

- Data Health 分为纯检测函数和 DuckDB loop runner，方便单测注入坏 K 线。
- 检测覆盖缺口、重复 timestamp、时间栅格偏移、OHLC 价格关系异常、极端 close 跳变。
- 最近 90 天完整率阈值默认 99%，由 loop 参数传入，后续可接配置。
- `auto_halt_on.data_gap=true` 时，缺口/低覆盖率会让报告标记 `halt_required=true`。
- DuckDB runner 按 exchange/symbol/timeframe 分组读取 S2 落库数据并输出 JSON 报告。
- S3 只报告和标记 halt，不修改交易配置、不触发任何真实交易动作。
- CLI `cql-data-health` 可对本地 `data/processed/market.duckdb` 生成 `reports/data_health.json`。
- 测试覆盖注入缺口、重复 K 线、异常价格、以及真实 DuckDB loop 报告生成。
