# S0 — 仓库骨架与工具链

- 目标是先建立可验证的 Python 项目骨架，不引入任何真实交易副作用。
- 空 loop 入口放在 `crypto_quant_loop.loops.empty_loop`，只写 `loop-run-log.jsonl`。
- CLI 使用标准库 `argparse`，避免 S0 对外部运行时依赖过重。
- `pyproject.toml` 统一声明 pytest、ruff、mypy 与后续配置加载依赖。
- CI 分三段：lint、type、test，并增加 gitleaks secret-scan。
- 本地没有 gitleaks 二进制时，用 `scripts/secret_scan.py` 作为轻量兜底扫描。
- 所有后续业务目录先建包，保持接口边界稳定。
- S0 不读取交易所密钥、不调用交易所、不做任何网络副作用。
