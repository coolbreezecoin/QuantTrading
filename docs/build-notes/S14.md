# S14 — L2 live adapter scaffold, dry-run only

- Implement a ccxt-shaped live adapter that reuses `Order`, `Fill`, and paper broker state concepts.
- Keep `dry_run` as a hard gate; dry-run records intents and synthetic fills without calling order endpoints.
- Read credentials only from environment variable names declared in `config/exchanges.yaml`; missing keys degrade to no-op credentials.
- Provide server-side protective stop/OCO request builders and submission path, but dry-run blocks execution.
- Reconciliation fetches exchange open orders and positions through an injected client, then treats exchange state as canonical.
- Live dry-run fixture must run signal -> order -> fill -> position -> reconciliation and assert no real order is reachable.
- Do not approve strategies, change registry notional caps, disable dry-run, enable alert channels, or require real keys.
